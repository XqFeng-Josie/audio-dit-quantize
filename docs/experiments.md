## 1. 背景

我们研究：把一个大型语音生成模型压缩到 4-bit（W4A4 量化）时，**用哪些数据做"标定"效果最好**——并提出从任务特征（文本难度、说话人、prompt/生成区域、扩散时间步）出发**自动选择标定数据**的方法。

### 1.1 任务：零样本声音克隆 TTS

给模型两样东西：一段某说话人的**参考音频**（prompt，约几秒）+ 一句**目标文本**，模型直接合成"这个人说这句话"的音频——不需要针对该说话人训练。衡量好坏主要看两点：**说得对不对**（合成音频转回文字后错字率低）和**像不像这个人**（音色相似度高）。

### 1.2 被量化的模型：LongCat-AudioDiT

美团 LongCat 团队的开源扩散 TTS 模型（[arXiv:2603.29339](https://arxiv.org/abs/2603.29339)，HF: `meituan-longcat/LongCat-AudioDiT-{1B,3.5B}`），在 Seed 基准上是 SOTA 水平。关键构件：

| 构件 | 说明 |
|---|---|
| Wav-VAE | 把 24 kHz 波形压缩成"波形隐空间"序列（不经过 mel 频谱，这是该模型的核心创新） |
| DiT 主干 | Diffusion Transformer，在隐空间做扩散去噪。1B 版：24 层 block，隐维 1536；每 block = self-attn + cross-attn + FFN，AdaLN 调制 |
| 文本编码器 | UMT5-base，把目标文本编成 cross-attn 的条件 |
| 采样 | 流匹配 ODE，**16 步**去噪（nfe=16），APG 引导（强度 4.0） |

推理时序列由两段拼接：**prompt 区**（参考音频的隐向量，负责"像谁"）+ **生成区**（被去噪合成的部分，负责"说什么"）。这个双区结构 + 16 个去噪时间步，是本项目"任务特征"的主要来源。

**量化只针对 DiT 主干的 GEMM 线性层**（self_attn / cross_attn / ffn 的投影矩阵）；VAE、文本编码器、embedding 等保持原精度。

### 1.3 量化与"标定"

- **W4A4**：权重（Weight）和激活（Activation）都用 4-bit 整数表示。数值只剩 16 个档位，误差主要来自少数数值极大的"离群通道"。
- **PTQ（训练后量化）**：不重训模型，只用**少量数据**（标定集，本项目 32 条）跑前向，据此确定量化参数（缩放、裁剪阈值、旋转/变换矩阵等）。
- **标定数据的作用**：量化参数是对着标定数据的激活分布优化出来的。标定集覆盖不了真实使用场景的分布，量化模型就会在没见过的分布上出错。**"选哪 32 条数据"因此是一个自由度很大、但长期被随意处理的设计选择——这正是本项目的研究对象。**

## 2. 研究计划

### 2.1 研究问题

> 在音频 DiT 的 PTQ 中，能否**从模型和任务的角度自动选择标定数据**（替代随机抽取/手工启发式），带来可证实的收益——**质量**（超过随机分布均值）、**效率**（更小标定预算达到同等质量）或**鲁棒性**（避开最差随机抽签）三档主张至少其一？

### 2.2 贡献主张（2026-07-19 全面修订；原版主张"廉价代理分数选集合超过随机"已被 E2+P2+W1 三重负结果否决，原文见 git 历史）

1. **主张 A（解剖，已基本成形）**：音频 DiT W4A4 退化的任务轴各向异性全景（CER/SIM × 位宽 × 尺度 × 错误类型/条目类别），加上**首个把"数据方差 vs 标定实例方差"显式分解的测量协议**（逐条配对 + 污染分级 + 预注册判读 + 实例重复）。核心可迁移结论：余量集中于 hard 可懂度（本质是重复结构鲁棒性）；SIM 成本 8-bit 即现、与标定无关、随尺度缓解；zh 存在 4-bit 特有红利；**总量指标上的"选数据收益"大部分是实例噪声幻觉**。
2. **主张 B（方法主线：任务特征 × 模型结构）**：**各向异性引导的量化改进**——测得的敏感度地图指导精度与损失容量沿模型结构轴分配：推理侧结构化混合精度（时间步轴 step-axis + 模块轴 M1；同一标定实例内配对，**天然免疫实例噪声**），标定损失侧任务对齐（region 加权 M2、敏感度通道加权 M3）。
3. **主张 C（数据侧，诚实闭环）**：标定数据选择的系统性否定（三重负结果 + 尺度不转移）与两个幸存例外——**语言匹配因果链**（对照+剂量+不对称+饱和，全程预注册）与**重复类残余效应**；连带方法论警示：单实例、总量指标下的标定集比较在小效应领域不可信。

### 2.3 统一叙事主张（2026-07-19 定稿）

两个贡献不是两个孤立技巧，而是同一条原理的两个投影：

> **TTS 的量化退化沿任务轴各向异性**——内容可懂度(CER) vs 音色(SIM)、prompt 区 vs 生成区、早期 vs 晚期扩散步、常规文本 vs 混淆音素。因此标定应当把**数据分布**与**目标函数**同时对齐到任务的敏感度结构（*task-sensitivity-aligned calibration*）：选数据 = 对齐 P(标定数据)，加权 loss = 对齐 L(标定目标)，两者共享同一套敏感度测量基础设施（E2）。

这也是与近邻工作的分界线：LLM/图像扩散的标定研究没有双区、没有说话人空间、没有 SIM/CER 的任务指标分解。首轮 GATE-B 结果（§4.2）已给出各向异性的第一块证据：hard CER 对标定数据高度敏感，SIM 完全不敏感。

**2026-07-19 修订**：原则本身（退化各向异性 → 对齐敏感度结构）被后续全部实验加强，但"对齐 P(标定数据)"在**集合选择**层面被证伪（E2/P2/W1）——数据分布对齐只在**语言层面**成立（§4.12）。叙事重心移到"对齐"的另外两个投影：**结构轴精度分配**（时间步/模块，推理侧）与 **L(标定目标) 对齐**（region/通道，损失侧）——即 §2.2 修订版主张 B。

结论：音频 DiT/TTS 上的任务感知标定（数据 + 目标）仍是空白；"标定加权"大方向在升温，**语音任务结构是护城河**。

### 2.6 量化方法基线与冻结原则（重要纪律）

改进必须建立在强基线上：FlatQuant 冻结为论文最优配置（详表见 `docs/paper-best-alignment.md`），SVDQuant 为第二基线（验证选择方法与量化方法正交），QuaRot 仅作方法选型表参考。**做数据选择实验时，一切贡献旋钮（loss 加权、region 权重）保持关闭**；增益只作为冻结基线之上的增量报告。

| 方法 | 类型 | 角色 | 入口 |
|---|---|---|---|
| FlatQuant | 梯度训练（逐 block 重建 + 可学习 Kronecker 变换 + LWC/LAC/diag） | **主基线**（唯一有显式 loss、对标定数据敏感的方法） | `flatquant_best.py` / `scripts/benchmark/benchmark_flatquant_best_seedtts.sh` |
| SVDQuant | 闭式（低秩分支 + 平滑网格搜索） | 第二基线（泛化性验证） | `svdquant_pipeline.py` / `scripts/benchmark/benchmark_svdquant_seedtts.sh` |
| QuaRot(-GPTQ) | 免训练（Hadamard 旋转 + GPTQ） | 方法选型表参考 | `quarot_linear.py` / `scripts/benchmark/benchmark_quarot_gptq_seedtts.sh` |

FlatQuant 标定流程：32 条标定数据各跑一次全精度推理（16 步 ODE），钩取 block-0 输入，每条沿轨迹取 2 个时间步快照（共 64 条序列）→ 逐 block 训练量化参数（200 步 × mb4，块输出 MSE）→ 冻结。

## 3. 实验计划

### 3.1 评测协议

**测试集**：Seed-TTS 官方测试集，三个子集——zh（中文常规，2020 条）、en（英文常规，1088 条）、hard（中文困难：绕口令/生僻组合，400 条）。

**指标**（都是"量化后 vs 全精度"越接近越好）：

| 指标 | 含义 | 工具 | 方向 |
|---|---|---|---|
| CER / WER | 合成音频经 ASR 转文字后的字/词错误率——**"说得对不对"** | Paraformer(zh) / Whisper(en) | ↓ |
| SIM | 合成音频与参考说话人的音色相似度（说话人向量余弦）——**"像不像这个人"** | WavLM 微调模型 | ↑ |
| MOS | 无参考音质分（已移出默认评测，wav 保留可事后补） | UTMOS + DNSMOS | ↑ |

**配对协议**：逐条固定种子（seed = base + 列表内条目序号），量化前后逐条严格配对；结论必须过 `paired_bootstrap.py` 的配对 bootstrap 置信区间。标定自身用 `--calib_seed` 固定。⚠ dev 列表索引 ≠ 全集索引——dev 运行只与其他 dev 运行（含 `p0_fp32` 参照）逐条配对，绝不与旧全集运行配对。

### 3.2 数据治理（本项目的核心协议）

**三层严格分离**：

| 层 | 用途 | 要求 |
|---|---|---|
| 候选池（candidate pool） | 标定数据的来源，选择算法在此池内挑选 | 与测试集**说话人级不相交**（L2，见下） |
| dev 划分 | 选择信号 / 实验间比较。**已冻结（2026-07-18，seed=20260718）**：zh 300/2020 + hard 100/400 + en 200/1088。uid 清单入库 `data/splits/*_dev_uids.txt`（冻结语义：清单存在时 `dev_split.py` 只物化不重采样）；gen/eval 全链路以 `zh_dev/hard_dev/en_dev` 集名使用 | 测试集的固定子集，仅用于开发期 |
| untouched test | 最终报告（集名 `*_heldtest`：zh 1720 / hard 300 / en 888） | 测试集其余部分，报告前绝不触碰 |

**污染等级定义**（数据选择研究必须 L2；方法对比允许 L1 + 记录 caveat）：

- **L0 配对级**：无标定条目与测试条目 (prompt, 文本) 完全相同；
- **L1 成分级**：prompt 音频、目标文本均不与测试集共享；
- **L2 说话人级**：prompt 说话人来自与测试集不同的语料库。

#### 3.2.1 候选池 v1 规格（2026-07-18，实现：`calib/pool.py` 构建 / `calib/audit.py` 审计）

**测量依据**（决定各筛选窗口）：Seed-TTS 测试 prompt 时长 zh p10/p50/p90 = 4.1/4.5/5.5s（几乎全部 ≥4.0s）、en = 3.4/4.5/6.0s；目标文本长度 zh 常规 18–28 字、hard 中位 51 字长尾至 385、en 41–86 字符。AISHELL-3 单句中位仅 3.2s → zh prompt 只取 ≥4s 的长句（≥4s 供给约 1.3 万条，无需拼接）；模型侧 `load_audio` 自动重采样，但池子仍写出裁剪后 24k 单声道副本（去首尾静音 top_db=35、自包含、路径稳定）。

**构成**（`pool_v1`，seed=20260718，共 376 条 / 158 说话人，旋钮均可调）：

| 子集 | 条数 | prompt 来源 | 目标文本来源 | 文本长度窗 |
|---|---|---|---|---|
| zh_normal | 240（120 说话人 × 2） | AISHELL-3，裁剪后 4.0–9.5s，性别轮转分层，每人取最接近 4.5s 的句子 | AISHELL-3 转写（**非 prompt 说话人**的句子，自带拼音）+“。” | 15–35 字 |
| zh_hardlike | 60 | 复用 zh prompt（打乱） | **按真实 hard 集分类学设计的确定性生成器**（三子风格，素材全部来自 AISHELL-3）：twister 词级混淆绕口令（高频二字词 + 同音/z-zh/n-l/in-ing 等混淆配对 + 句法模板，功能字停用表过滤）/ repeat 句子与片段重复 4–6 次 / concat 多句拼接长句；子风格记入元数据 `hard_substyle` | 25–120 字 |
| en_normal | 76（38 说话人 × 2） | LibriTTS dev-clean，裁剪后 3.0–9.5s | LibriTTS normalized 文本（非 prompt 说话人） | 40–90 字符 |

**交叉配对协议**（镜像测试集结构）：目标文本绝不来自 prompt 自身的句子或说话人；每个 prompt 音频至多用于一个 normal 条目（hardlike 可复用 prompt，审计仅提示不报错）；目标文本池内唯一。

**元数据表**（`pool_v1_meta.csv`，即 P1 任务侧特征矩阵）：说话人/性别/年龄段/口音（AISHELL-3 spk-info 免费提供）、prompt 时长与 RMS 响度、文本来源、字数、音节数、去调唯一音节数、最大同音节重复次数、唯一声母/韵母数、唯一字符比。

**审计政策**（`calib/audit.py`，不通过则池子不可用）：A 音频不得位于 seedtts_testset 内；B prompt 文件名不得与测试 prompt 碰撞；C 归一化文本不得与任何测试 prompt/目标文本相等；D ≥10 字的归一化文本不得与测试文本互为包含；E 池内卫生（uid/目标文本唯一、wav 存在、字段完整）。说话人级（L2）隔离由语料库来源保证（AISHELL-3/LibriTTS vs Seed-TTS 的 DiDiSpeech/Common Voice）并记录于此。

**抽样器**：`python -m audio_dit_quantize.calib.pool sample --pool data/calib_pool/pool_v1.lst --n 32 --seed <k> --out data/calib_pool/sets/rand32_s<k>.lst`（wav 路径按输出目录重写，`load_items` 直接可用）。

**已知局限（记录在案）**：① hardlike 为合成文本：twister 子风格的"词"来自 bigram 挖掘，仍有少量跨词边界的伪词（发音混淆属性正确，语义通顺性不保证）；真实 hard 集 max_char_run p90=6 的字符连跑尾部未覆盖；② en 无 hard 子集（测试集也没有）；③ AISHELL-3 转写无标点，构建时统一补句号；④ 语料域差异（朗读风格 vs 测试集口语域）未量化（原 E1 锚点实验已取消）。

### 3.3 实验推进与统计纪律（2026-07-19 确立）

**背景**：既有经验表明 FlatQuant paper-best 在常规集上已逼近 fp32——我们在**小效应量**领域作战，"很多方向表现不稳定"是统计功效不足的症状，不是玄学。所有推进决策遵守：

1. **余量地图先行**。任何新方向动工前，先算目标 (set, metric) 上的 gap = 量化 − fp32（配对 CI）。**gap 就是一切改进方法的收益上限**；gap ≈ 0 的指标不能区分标定集好坏，从选择/比较信号中剔除。（首轮实测见 §4.2：主战场 = hard CER，SIM 不可动。）
2. **散布 vs gap 的比较才算判读**。组间散布占总 gap 的比例回答"值不值得选"。（2026-07-19 用户决定：**不做 seed 重复对照**，全部按单 `calib_seed=0` 解读，标定实例噪声并入"换随机集"的变差一并处理，不单独剥离。）**同日补注**：后补做了 s0 × 3 实例判别（§4.8），确立实例噪声标尺——σ_inst(hard)≈0.18、σ_inst(重复类)≈0.19、两 run 之差噪声底 ±0.25；**此后一切单 run 小效应比较必须对照此标尺**，单点显著性可被实例噪声翻转（§4.8 实证）。**W1 修正（§4.11，3 集 × 3 实例）**：σ_inst(hard 总量)=0.239、σ_inst(重复类)=0.300、σ_inst(zh)=0.066；单 run 之差噪声底 hard ±0.34 / 重复类 ±0.42（95%）——旧标尺偏松，以此为准。
3. **设计对照优于回归淘金**。随机组的特征回归（样本少、特征多）只许用于提假设；确认一律用**单因子最大对比构造集**（全 hard vs 全 normal、多说话人 vs 单说话人、早 vs 晚 timestep 等），一次对照仅 2 个作业，功效远高于回归。
4. **优先做无空结果风险的实验**。预算悬崖扫描、最差抽签兜底这类"效应必然随某轴放大"的设计优先；追逐微小均值改善的方向（新 loss 花样等）必须先过设计对照的效应量门槛才继续投入。
5. 既有纪律不变：逐条配对 + paired bootstrap；冻结 base（§2.6）；dev 选、heldtest 报（§3.2）。

### 4.7 错误类型机制归因（离线分析）——2026-07-19

2. **screen-then-select 重定位**：按总 CER 筛选 = 筛噪声，路线 (b) 原形式作废；如仍要数据侧方法，唯一站得住的形式是**重复类探针 + 每候选 ≥2 实例**的筛选（成本 ~10 GPU·h/10 候选，价值有限，降为可选）。
3. **W3（说话人对照）降级待议**：其嫌疑相关（max_per_spk −0.49）是对总 hard CER 算的——噪声主导轴上的淘金，先验大幅走弱（对重复类仅 −0.25）；预授权的发车条件（"W1 确认数据效应"）仅在重复类窗口部分成立，**建议暂缓，等用户决策**。
4. **M 系列升格为主线**：结构侧/损失侧方法不受本判决影响，且 M1（cross-attn 混合精度）的机制假设恰好指向幸存的重复类轴——数据侧修不动的，结构侧来修。
5. 语言匹配（zh 轴）幸存：修正后 σ_inst(zh)=0.066，en12−en0 的 +0.255 ≈ 2.7σ 仍过线；W2（en32 剂量）为最终仲裁。
6. 判读标尺更新（§3.3 同步）：单 run 之差噪声底 hard 总量 ±0.34、重复类 ±0.42（95%）——比 §4.8 旧标尺更严。

### 5.3 命令速查

- 环境：`source env.sh`
- Phase 0：`GPUS=... K=10 SEED_REPEATS="" bash scripts/calib/phase0_sensitivity.sh 1b`（幂等，失败后原命令续跑；seed 对照 `SEED_REPEATS="1 2"`）
- INT8 参照档：`GPUS=... EVAL_METRICS="wer cer sim" bash scripts/benchmark/benchmark_int8_seedtts.sh 1b "zh_dev,hard_dev,en_dev"`
- MOS 事后补：`bash scripts/evaluate_seedtts_metrics.sh <gen_root> <tag> "zh_dev hard_dev en_dev" mos`
- 配对显著性：`python -m audio_dit_quantize.paired_bootstrap --metric cer|sim --a <fp32结果> --b <方法结果>`
- 错误类型归因（离线、零 GPU）：`python -m audio_dit_quantize.calib.error_taxonomy`；新集合复验：`--extra_runs p2_rule_r0 p2_rule_r1`（自动跑 §4.7 预注册检验）
- en SIM 补评（wav 保留的 run）：`GPUS=<卡> bash scripts/evaluate_seedtts_metrics.sh gen/paired/<tag> <tag> en_dev sim`
- E2 探针（一次性）：`python -m audio_dit_quantize.calib.pool probe --pool data/calib_pool/pool_v1.lst`（CPU 建集）→ `python -m audio_dit_quantize.calib.sensitivity.probe_capture --probe_lst data/calib_pool/probe/probe_v1.lst --out data/calib_pool/probe/probe_capture.pt`（GPU ~5 min）
- E2 打分：`python -m audio_dit_quantize.calib.sensitivity.score --calib_lst <lst> --out data/calib_pool/scores_v2/<名字> --probe_capture data/calib_pool/probe/probe_capture.pt`（单集 ~15 min GPU；不带 --probe_capture 则只算 v1 分数）；校验：`python -m audio_dit_quantize.calib.sensitivity.validate --scores_dir data/calib_pool/scores_v2`
- 基线冒烟：`LIMIT=1 bash scripts/benchmark/benchmark_flatquant_best_seedtts.sh 1b hard`
